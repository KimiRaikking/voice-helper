〇、快速开始(跨平台,Git 仓库)

任意 Python 3.10+,安装脚本自建 .venv 并装好对应平台依赖、配置开机自启。

git clone https://github.com/KimiRaikking/voice-helper.git
cd voice-helper
python install.py          # Windows 也可用 py install.py
  • Windows装 faster-whisper(CPU)+ SenseVoice,启动文件夹放隐藏启动项,装完即生效。手动启动 run.bat,看报错 run-debug.bat。
  • macOS装 mlx-whisper(GPU)+ SenseVoice,配 LaunchAgent。额外需系统设置授权(第⑧节)。
  • 改配置:编辑 voice.env 后重启服务(第⑥/⑦节)。
首次说话会自动下载模型(SenseVoice 从 ModelScope,Whisper 从 HuggingFace)。

① 它能做什么

  • 全局可用:任何能接受粘贴的输入框都能用。
  • 按住说话:按住热键(默认右 Option / 右 Alt)录音,松开转写并自动粘贴。
  • 多引擎可切换:Whisper / SenseVoice,VOICE_ENGINE 一键切换,模型本地、互不替换。
  • 状态可见:mac 菜单栏 / Windows 系统托盘图标,标注当前引擎。
  • 强制简体中文、开机自启(崩溃自重启)。
例外:密码输入框(系统安全键盘屏蔽模拟粘贴)、全屏独占键盘的 App 可能用不了。

③ 日常使用

  1. 光标放进任意输入框。
  2. 按住热键(默认右 Option / 右 Alt)→ 说话 → 松开。
  3. 文字自动转写并粘贴。
  • macOS图标在屏幕最顶部菜单栏(刘海可能挡住)。
  • Windows图标在右下角系统托盘(被折叠则点 ^ 展开);右键看菜单。

所有配置写在 voice.env,voiced 启动时自动读取,两平台通用。

⚙️ 配置优先级:voice.env 是唯一配置源(mac 的 LaunchAgent plist 已不再写环境变量);进程环境里真设了同名变量则优先于 voice.env。

⑥ 如何修改配置

通用:编辑 voice.env → 保存 → 重启服务。

macOS

open -e ~/voice-helper/voice.env
launchctl kickstart -k gui/$(id -u)/com.voicehelper.dictation

Windows(仓库目录,双击 .bat 即可)

notepad voice.env
restart.bat      :: 重启生效(= 停止 + 启动)

⑦ 开机自启 / 服务管理

macOS(LaunchAgent)

launchctl list | grep voicehelper                                # 是否在跑
launchctl kickstart -k gui/$(id -u)/com.voicehelper.dictation     # 重启
launchctl unload ~/Library/LaunchAgents/com.voicehelper.dictation.plist  # 关闭
launchctl load   ~/Library/LaunchAgents/com.voicehelper.dictation.plist  # 开启
tail -f ~/voice-helper/voiced.log                                # 看日志

Windows(启动文件夹 VBS)

仓库带一键脚本(双击即可):status.bat(是否在跑+日志)· restart.bat(重启)· stop.bat · run.bat(手动启动)· run-debug.bat(前台看报错)。

status.bat       :: 是否在跑 + 日志末尾
restart.bat      :: 重启
del "%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\voice-helper.vbs"  :: 关闭自启
调试用 run-debug.bat(显示控制台、实时报错),Ctrl+C 退出。

⑧ 系统权限

macOS(必做)

给那个 Python 二进制(.venv/bin/python 解析后的真实路径)授三项权限:

权限用途
输入监控监听热键
辅助功能模拟 Cmd-V 粘贴
麦克风录音
系统设置→隐私与安全性逐项添加;文件框灰掉就从 Finder 拖拽该二进制。授权后重启服务。

Windows(基本无需)

  • 不需要输入监控/辅助功能类授权。
  • 首次录音若弹麦克风询问,允许;或 设置→隐私→麦克风→「允许桌面应用访问麦克风」。
  • SmartScreen/杀软提示未知发布者时选「仍要运行」。

⑨ 常见问题 / 诊断排查

⚡ 一键诊断(出问题先跑这个)

Git Bash,仓库目录:

bash voicectl.sh doctor

输出 8 行(200=通,000=连不上)+ 日志最后报错:venv / 运行 / 引擎 / SenseVoice模型 / Paraformer模型 / 连GitHub / 连ModelScope / 连HuggingFace。

🔍 手动逐项检查

# venv 是否装好
ls ~/voice-helper/.venv/Scripts/python.exe

# SenseVoice 模型是否真下好(总大小约 900M~1G,含 model.pt)
du -sh ~/.cache/modelscope/hub/models/iic/SenseVoiceSmall
ls -lh ~/.cache/modelscope/hub/models/iic/SenseVoiceSmall

# Paraformer 模型
du -sh ~/.cache/modelscope/hub/models/iic/speech_seaco_paraformer*

# 运行状态 + 日志
bash voicectl.sh status
tail -n 30 ~/voice-helper/voiced.log

# 网络连通(200=通,000=连不上/被代理拦)
curl -s -m 8 -o /dev/null -w "%{http_code}\n" https://www.modelscope.cn
curl -s -m 8 -o /dev/null -w "%{http_code}\n" https://huggingface.co

🏢 公司网络:连接全 000 / authentication required

需要走带认证的代理。但:
  • 日常 SenseVoice 不需要网络——模型下好(约 900M)后纯本地运行,断网也能用,直接按热键说话测试。
  • 让 SenseVoice 彻底离线:voice.env 指向本地目录
    SENSEVOICE_MODEL=C:\Users\<用户名>\.cache\modelscope\hub\models\iic\SenseVoiceSmall
    再 bash voicectl.sh restart。
  • 下模型 / git 时配代理:
git config --global http.proxy http://用户:密码@IP:端口
export HTTPS_PROXY=http://用户:密码@IP:端口
export HTTP_PROXY=http://用户:密码@IP:端口
bash voicectl.sh restart

常见现象

(Windows)怎么确认装好/生效了? 仓库目录双击 status.bat:显示进程是否在跑(running, PID…/NOT running)+ voiced.log 末尾。再看右下角托盘图标;最后按住右 Alt 说句话出字即生效。显示 NOT running → 双击 run.bat,看报错用 run-debug.bat。

按了没反应 / 录音失效?
mac 日志报 PaErrorCode -9986 = 音频设备变了,重启服务 launchctl kickstart -k gui/$(id -u)/com.voicehelper.dictation。
win 切换音频设备后双击 restart.bat 重启。还不行查权限。

图标看不到? mac 在顶部菜单栏(刘海挡);win 在右下角托盘(点 ^ 展开)。

有识别没粘贴? mac 缺辅助功能权限;win 目标窗口可能拦截模拟键,文字已在剪贴板可手动 Ctrl+V。

输出繁体? 已用 OpenCC 转简体;确认 opencc 在 venv 里。

改了不生效? 重启服务(第⑥/⑦节)。

下载 Whisper 大模型卡 0.00B? HF 改用 Xet,国内常卡死。先试禁用 Xet:

HF_HUB_DISABLE_XET=1 ~/voice-helper/.venv/bin/python -c "from huggingface_hub import snapshot_download; print(snapshot_download('mlx-community/whisper-large-v3-mlx'))"

large-v3 权重 weights.npz ~3GB 仍卡死时(国内网络对 huggingface_hub 客户端的 3GB 文件极不稳),用最可靠的办法:curl 国内镜像拉到本地目录(纯 HTTP + 断点续传):

DIR=~/voice-helper/models/whisper-large-v3-mlx
mkdir -p "$DIR" && cd "$DIR"
curl -L -C - --retry 30 --retry-delay 5 --retry-all-errors -o weights.npz \
  "https://hf-mirror.com/mlx-community/whisper-large-v3-mlx/resolve/main/weights.npz"
curl -L -o config.json \
  "https://hf-mirror.com/mlx-community/whisper-large-v3-mlx/resolve/main/config.json"

再把 voice.env 的 WHISPER_MODEL 指向这个本地目录(mlx 支持本地路径):WHISPER_MODEL=/Users/<你>/voice-helper/models/whisper-large-v3-mlx。models/ 已 gitignore。

SenseVoice 走 ModelScope(国内快),一般不受影响——中文场景直接用 SenseVoice 引擎通常更省事。

⑩ 热词偏置:动态喂词(治技术同音词)✅

paraformer 引擎用 SeACo-Paraformer 做热词偏置:给一份词清单,遇同音歧义强制优先匹配。模型从 ModelScope 下载(国内快)。

启用:voice.env 设 VOICE_ENGINE=paraformer → 重启。

动态喂词(核心):热词放 hotwords.txt(每行一个),引擎每次转写实时读取,加词无需重启。工作流:说一句 → 某词转错 → 立刻喂正确词 → 再说就对了。

addhot 时延 推理 吞吐量            # 加词(任意目录)
python ~/voice-helper/add_hotword.py 时延 推理   # 等价
addhot                            # 查看当前热词
  • 静态热词也可写 voice.env 的 VOICE_HOTWORD=时延 推理(与 hotwords.txt 合并)。
  • hotwords.txt 是个人词库,已 gitignore;仓库带起步样例 hotwords.txt.example。
  • 跨平台通用(Windows 用 python add_hotword.py …)。
注:Paraformer 输出不带标点(SenseVoice 带);重术语准确度选 Paraformer,重标点选 SenseVoice。

⑪ 字词自动纠正(所有引擎通用)✅

转写后套一张纠正表做替换,适合 AI agent:某词总被转错,登记一条规则,以后自动纠正。对 SenseVoice / Whisper / Paraformer 都生效。规则在 corrections.txt(每行 错词=>对词),每次转写实时读取,即改即生效。 
addfix 食盐 时延                 # 加规则(任意目录)
python ~/voice-helper/add_fix.py 食盐 时延   # 等价
addfix                          # 查看所有规则
# Windows Git Bash:
bash voicectl.sh fix 食盐 时延
长词优先匹配避免误伤;corrections.txt 个人表已 gitignore(样例 corrections.txt.example)。三层叠加最稳:Paraformer 热词(转写时)→ 自动纠正表(转写后)→ 简繁转换。
